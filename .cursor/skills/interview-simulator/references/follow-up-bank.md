# Follow-Up Question Bank

Categorized follow-up questions for interview simulation, organized by problem type.

## Universal Follow-Ups (apply to any problem)

- "Walk me through your approach before you start coding."
- "What's the time complexity? Space complexity?"
- "What are the edge cases?"
- "Can you think of an alternative approach?"
- "What trade-offs does your solution make?"
- "How would you test this?"

## Array / String Problems

- "What if the array is empty?"
- "What if there's only one element?"
- "What if all elements are the same?"
- "What if the array is already sorted?"
- "Does your solution work with negative numbers?"
- "What if the input doesn't fit in memory? How would you handle it?"
- "Can you do this in-place without extra space?"
- "What if we need to preserve the original order?"

## Two Pointers / Sliding Window

- "Why does the two-pointer approach work here? Can you prove correctness?"
- "What invariant does your window maintain?"
- "What happens when you shrink vs. expand the window?"
- "Can you handle the case where no valid window exists?"
- "What if the constraint changes from 'exactly K' to 'at most K'?"

## Tree / Graph Problems

- "What if the tree is unbalanced? How does that affect complexity?"
- "Would BFS or DFS be better here? Why?"
- "What if there are cycles in the graph?"
- "How do you handle disconnected components?"
- "Can you do this iteratively instead of recursively?"
- "What's the space complexity of your recursion? Can you reduce it?"

## Dynamic Programming

- "How did you define your state?"
- "Can you explain the recurrence relation?"
- "What's the base case and why?"
- "Can you optimize the space from O(n^2) to O(n)?"
- "Would top-down or bottom-up be better here? Why?"
- "How would you reconstruct the actual solution (not just the optimal value)?"
- "What if we need the K-th best solution instead of the best?"

## Binary Search

- "Why is binary search applicable here?"
- "How do you handle the boundary conditions (left vs. right inclusive)?"
- "What property makes the search space monotonic?"
- "What if there are duplicates?"
- "Can you identify the invariant maintained at each step?"

## Backtracking

- "What's your pruning strategy?"
- "How do you avoid generating duplicates?"
- "What's the worst-case number of recursive calls?"
- "Can you convert this to an iterative approach?"
- "What if we added a constraint — how would you prune earlier?"

## Greedy

- "Can you prove your greedy choice is correct?"
- "Give me a counterexample where a different greedy strategy fails."
- "What if the greedy property doesn't hold — what algorithm would you use instead?"
- "Is there a case where this problem requires DP instead of greedy?"

## Hash Map / Counting

- "What if we can't use extra space?"
- "How do you handle hash collisions in a real implementation?"
- "What if the input stream is infinite?"
- "Can you solve this with a single pass?"

## Linked List

- "Can you do this without extra space?"
- "How do you handle the case where the list has a cycle?"
- "What if the list is doubly linked — does that change your approach?"
- "Can you do this in one pass?"

## Complexity Challenge Follow-Ups

- "Your solution is O(n^2). The interviewer says there's an O(n log n) solution. How would you approach it?"
- "Can you reduce from O(n) space to O(1)?"
- "What if n goes from 10^5 to 10^9? Does your approach still work?"
- "What's the theoretical lower bound for this problem? Is your solution optimal?"
